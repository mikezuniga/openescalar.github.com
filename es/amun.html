Amun IaaS Cloud Management

Es la primera parte de la aplicación OpenEscalar, diseñado para permitir a los usuarios administrar y coordinar IaaS desde una interfaz web central. Amun ofrece las siguientes características:

  • Gestión de nubes públicas y privadas.
  • Gestión de usuarios.
  • Orquestación.
  • Agrupar instancias por roles y definir las tareas propias de cada rol, lo que permite al usuario ejecutar tareas por servidor, rol, siguiendo flujos de trabajo o despliegues.
  • Monitoreo.
  • Auto Scaling
  • Registro Central.
  • Gráficos.
  • Completamente Distribuida.
  • DSL, que permiten a los usuarios crear infraestructuras en un lenguaje de alto nivel.


Arquitectura


Componentes

Amun está formado por los siguientes componentes:

  • Amun WebUI

    Aplicación de Ruby on Rails que permite a los usuarios interactuar con el marco de Amun, además de ejecutar trabajos, comprobar los registros, organizar y crear gráficos de control.

  • Amun Tools

    Amun Tools es un conjunto de procesos que proporcionan los siguientes servicios:

    • Servidor Api :
      Descifra y decodifica todos las peticiones que vienen de los clientes, y utiliza un triple factor de encriptación para proveer las tareas requeridos por los clientes.
    • oe-commander:
      Recoge estadísticas de los clientes, define los limites para autoescalar y aumenta o reduce las infraestructuras.
    • oe-coordinador:
      Coordina y distribuye a todos los trabajadores oe todas las solicitudes enviadas ya sea por los usuarios a través de WebUI Amun o oe-commander, que utiliza un balanceo de carga interna para mantener a los trabajadores de la sobrecarga de solicitudes múltiples.
    • oe-worker:
      Realiza todas las peticiones para crear o destruir volúmenes, servidores de seguridad, instancias keyparis y direccion IP a las clouds, y actualiza la base de datos con detalles como numero de serie, dirección IP, hosts name, tamaños después de obtener una respuesta de la cloud.
  • Cliente Amun

    El cliente, el cual es instalado en cada maquina virtual, se autoentifica con el servidor para solicitar las tareas indicadas por Amun. Una vez que las obtiene, las ejecutará una a la vez, recogera el código de salida, el resultado actualizar Amun con ellos. El cliente se conecta y comprueba si hay nuevas notificaciones en el servidor de cola de mensajes cada minuto.

  • Servidor de cola de mensaje

    Servicio al que Amun WebUI envía notificaciones las cuales son recibidas por los clientes en las instancias de máquina virtual. Cada notificación contiene información suficiente para que el cliente pueda solicitar al servidor API la tarea correcta.

  • Servidor de registro

    Servicio de registro central al cual todos los clientes que envían sus registros los cuales pueden ser vistos desde el WebUI Amun.
